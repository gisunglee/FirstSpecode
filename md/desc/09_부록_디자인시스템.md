# Chapter 9 (부록). 디자인 시스템 — `sp-*` 와 CSS 토큰

> **이 챕터의 정체**: 우리 프로젝트의 UI 작성 **절대 규칙**.
> 처음에는 "이거 왜 이렇게 해?" 싶지만, 이걸 어기면 **테마 전환과 일관성이 깨진다.**
> Java 비유: Spring 의 `application.yml` 으로 환경값 분리하는 것과 같은 의도. **하드코딩 금지** 와 동일.

---

## 9-1. 핵심 규칙 3개 (절대)

[.claude/CLAUDE.md](../../.claude/CLAUDE.md) 의 "UI 작업 절대 규칙" 그대로:

1. **하드코딩 금지** — 색상/간격/반경/폰트는 semantic 토큰 (`--color-*`, `--space-*`, `--radius-*`, `--text-*`) 사용
2. **`sp-` prefix** — 모든 컴포넌트 클래스는 `sp-` 시작, 상태는 `is-` prefix
3. **`data-theme`** — 테마 전환은 `document.documentElement.setAttribute('data-theme', 'dark|light|dark-purple')` 로만

---

## 9-2. 왜 토큰을 쓰는가

### ❌ 하드코딩 — 우리 프로젝트에서 금지
```tsx
<div style={{ background: "#1e1e1e", color: "#e0e0e0" }}>...</div>
//                       ^^^^^^^^^^         ^^^^^^^^^^
//                       다크 테마에서만 OK. 라이트 테마 전환 시 깨짐.
```

### ✅ 토큰 사용
```tsx
<div style={{
  background: "var(--color-bg-card)",
  color:      "var(--color-text-primary)"
}}>...</div>
```

테마가 바뀌어도 같은 토큰 이름이 다른 색으로 매핑되므로 자동 전환.

---

## 9-3. CSS 변수 (Custom Property) — Java 개발자 시점

**이건 React 개념이 아니라 CSS 자체 기능이다.**

```css
/* 정의 */
:root {
  --color-bg-card: #ffffff;
}

[data-theme="dark"] {
  --color-bg-card: #1e1e1e;     /* 같은 이름, 다른 값 */
}

/* 사용 */
.my-card {
  background: var(--color-bg-card);
}
```

### 동작 원리
- HTML 의 `<html data-theme="dark">` 속성을 바꾸면 → 변수 값이 변경됨 → 모든 사용처에 자동 반영
- **JavaScript 가 개입할 필요 없음**. CSS 가 알아서 한다.

> Spring 의 `@Value("${color.card}")` + `application-dark.yml` / `application-light.yml` 프로파일 전환과 의도가 같다.
> 다만 CSS 변수는 **런타임에 즉시 전환** 가능 (서버 재시작 없음).

## 9-5. `sp-` 클래스 — 컴포넌트 단위

토큰을 직접 쓰는 것보다, **`sp-*` 클래스 한 줄**로 끝내는 게 우선.

### 입력 폼
```tsx
<label className="sp-label">프로젝트명</label>
<input className="sp-input" placeholder="입력하세요" />
<select className="sp-input">...</select>
<textarea className="sp-input">...</textarea>
```

### 버튼
```tsx
<button className="sp-btn sp-btn-primary">저장</button>
<button className="sp-btn sp-btn-secondary">취소</button>
<button className="sp-btn sp-btn-danger">삭제</button>
```

### 상태 클래스 (`is-` prefix)
```tsx
<button className="sp-btn sp-btn-primary is-loading">처리 중...</button>
<input  className="sp-input is-error" />
```

### 우리 프로젝트 실전 — [src/app/(main)/projects/page.tsx:131-184](../../src/app/(main)/projects/page.tsx#L131-L184)
```tsx
<label className="sp-label">프로젝트명 <span style={{ color: "var(--color-error)" }}>*</span></label>
<input className="sp-input" value={name} onChange={(e) => setName(e.target.value)} autoFocus />

// ...
<button type="button" className="sp-btn sp-btn-secondary" onClick={onClose}>취소</button>
<button type="submit" className="sp-btn sp-btn-primary" disabled={mutation.isPending}>생성</button>
```

> CSS 정의는 [.claude/design/components.css](../../.claude/design/components.css) 에 있다. 직접 편집할 일은 거의 없음.

## 9-6. inline style 도 토큰을 쓴다

복잡한 레이아웃은 inline style 도 쓴다 (우리 프로젝트 패턴). **단 안의 값은 토큰으로.**

```tsx
<div
  style={{
    background:   "var(--color-bg-card)",
    border:       "1px solid var(--color-border)",
    borderRadius: "var(--radius-card)",
    padding:      "var(--space-4)",
    boxShadow:    "var(--shadow-md)",
  }}
>
  ...
</div>
```

> ⚠️ **`#ffffff`, `12px`, `rgba(0,0,0,0.4)` 같은 직접 값이 들어가는 순간 규칙 위반.**
> 단 1개 예외: 모달 오버레이의 `rgba(0,0,0,0.4)` 같은 임시값은 허용 (현재 코드에 존재).

## 9-7. 테마 전환

### 변경
```tsx
import { useAppStore } from "@/store/appStore";

function ThemeToggle() {
  const { theme, setTheme } = useAppStore();
  return (
    <button onClick={() => setTheme(theme === "dark" ? "light" : "dark")}>
      테마 전환
    </button>
  );
}
```

`setTheme` 안에서 `document.documentElement.setAttribute("data-theme", theme)` 가 실행됨 ([src/store/appStore.ts:76-82](../../src/store/appStore.ts#L76-L82)).

### 3가지 테마
- `dark` (기본)
- `light`
- `dark-purple`
