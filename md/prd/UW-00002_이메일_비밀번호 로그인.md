#### FID-00014 화면 진입 시 자동 처리

#### 기능: [FID-00014] 화면 진입 시 자동 처리

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | SELECT |
| **API** | `POST /api/auth/token/refresh` |
| **트리거** | 화면 진입 시 자동 실행 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| refreshToken | string | N | 저장된 Refresh Token (없으면 자동 로그인 스킵) |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| accessToken | string | 갱신된 JWT Access Token |
| refreshToken | string | 갱신된 Refresh Token |

**처리 로직**
```
1. 로컬스토리지에서 아이디 저장 값 확인 → 있으면 이메일 필드 자동 입력 및 체크박스 체크
2. 자동 로그인 Refresh Token 존재 여부 확인
3. RT 존재 시 POST /api/auth/token/refresh 호출
4. 서버 처리:
   a. tb_cm_refresh_token에서 token_hash_val 조회
   b. revoke_dt IS NULL AND expiry_dt > NOW() 확인
   c. 유효 시: 새 AT/RT 발급, 기존 RT revoke_dt = NOW() 처리
   d. auto_login_yn = 'Y'이면 RT expiry_dt 연장 (10일)
5. 성공 시 토큰 저장 후 대시보드 이동
6. 실패(만료/무효) 시 저장된 RT 삭제, 로그인 폼 표시
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_refresh_token | token_hash_val | RT 해시 조회 |
| tb_cm_refresh_token | expiry_dt | 만료 여부 확인 |
| tb_cm_refresh_token | revoke_dt | 폐기 여부 확인 (NULL이면 유효) |
| tb_cm_refresh_token | auto_login_yn | 자동 로그인 시 RT 연장 여부 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| RT 만료/무효 | 401 | (무시) 로그인 폼 표시 |

#### FID-00015 로그인 실행

#### 기능: [FID-00015] 로그인 실행

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/login` |
| **트리거** | [로그인] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| email | string | Y | 로그인 이메일 |
| password | string | Y | 비밀번호 |
| rememberMe | boolean | N | 자동 로그인 여부 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| accessToken | string | JWT Access Token |
| refreshToken | string | JWT Refresh Token |
| failCount | number | 실패 시 연속 실패 횟수 (성공 시 미포함) |
| lockExpiredAt | string | 잠금 시 해제 예정 시각 ISO 8601 (잠금 시에만 포함) |

**처리 로직**
```
1. 이메일·비밀번호 공백 검증
2. 로그인 API 호출
3. 서버 처리:
   a. tb_cm_member에서 email_addr로 회원 조회
   b. tb_cm_account_lock에서 활성 잠금 확인 (lock_sttus_code = 'LOCKED' AND lock_expiry_dt > NOW())
      → 잠금 상태면 423 응답 (lockExpiredAt 포함)
   c. pswd_hash와 bcrypt 비교
   d. 실패 시:
      - tb_cm_login_attempt INSERT (succes_yn = 'N', fail_rsn_cn = 'INVALID_PASSWORD')
      - 최근 연속 실패 횟수 카운트 (tb_cm_login_attempt에서 해당 회원의 마지막 성공 이후 실패 건수)
      - 5회 도달 시: tb_cm_account_lock INSERT (lock_expiry_dt = NOW() + 1시간)
      - 401 응답 (failCount 포함, 5회면 423으로 lockExpiredAt 포함)
   e. 성공 시:
      - tb_cm_login_attempt INSERT (succes_yn = 'Y')
      - JWT Access Token 발급
      - tb_cm_refresh_token INSERT (auto_login_yn = rememberMe ? 'Y' : 'N', expiry_dt = NOW() + 10일)
      - tb_cm_member_session INSERT (device_info_cn, ip_addr)
4. 클라이언트:
   a. 성공 시 토큰 저장, 아이디 저장 체크 시 이메일 로컬 저장
   b. 대시보드로 이동
   c. 401 시 실패 메시지 표시 (failCount 포함)
   d. 423 시 로그인 폼 비활성화 → 계정 잠금 안내 영역 표시
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | email_addr | 회원 조회 |
| tb_cm_member | pswd_hash | 비밀번호 bcrypt 비교 |
| tb_cm_login_attempt | attempt_id | PK, UUID 자동 생성 |
| tb_cm_login_attempt | mber_id | 로그인 시도 회원 FK |
| tb_cm_login_attempt | attempt_ip_addr | 클라이언트 IP |
| tb_cm_login_attempt | succes_yn | 'Y' 또는 'N' |
| tb_cm_login_attempt | fail_rsn_cn | 실패 사유 (INVALID_PASSWORD 등) |
| tb_cm_account_lock | lock_id | PK, UUID 자동 생성 (5회 실패 시) |
| tb_cm_account_lock | mber_id | 잠금 대상 회원 FK |
| tb_cm_account_lock | lock_rsn_cn | '5회 연속 로그인 실패' |
| tb_cm_account_lock | fail_cnt | 5 |
| tb_cm_account_lock | lock_expiry_dt | NOW() + 1시간 |
| tb_cm_account_lock | lock_sttus_code | 'LOCKED' 초기값 |
| tb_cm_refresh_token | token_id | PK, UUID 자동 생성 |
| tb_cm_refresh_token | mber_id | 로그인 회원 FK |
| tb_cm_refresh_token | token_hash_val | RT 해시 저장 |
| tb_cm_refresh_token | auto_login_yn | rememberMe 반영 |
| tb_cm_refresh_token | expiry_dt | NOW() + 10일 |
| tb_cm_member_session | sesn_id | PK, UUID 자동 생성 |
| tb_cm_member_session | mber_id | 로그인 회원 FK |
| tb_cm_member_session | device_info_cn | User-Agent 등 |
| tb_cm_member_session | ip_addr | 클라이언트 IP |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 이메일·비밀번호 불일치 | 401 | 이메일 또는 비밀번호가 올바르지 않습니다 (N회 실패) |
| 5회 실패 → 잠금 진입 | 423 | 5회 연속 실패로 계정이 잠겼습니다. HH:mm 이후 다시 시도하거나 잠금 해제 메일을 요청해 주세요 |
| 이미 잠금 상태에서 시도 | 423 | 계정이 잠금 상태입니다. HH:mm 이후 다시 시도하거나 잠금 해제 메일을 요청해 주세요 |
| 미인증 계정 | 403 | 이메일 인증이 완료되지 않았습니다 |
| 공백 입력 | - | 이메일/비밀번호를 입력해 주세요 |
| 서버 오류 | 500 | 로그인 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |

#### FID-00017 잠금 해제 메일 발송

#### 기능: [FID-00017] 잠금 해제 메일 발송

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/unlock/email` |
| **트리거** | [잠금 해제 메일 발송] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| email | string | Y | 잠금된 계정 이메일 |

**처리 로직**
```
1. 서버에 잠금 해제 메일 발송 API 호출
2. 서버 처리:
   a. tb_cm_member에서 email_addr로 회원 조회
   b. tb_cm_account_lock에서 활성 잠금 건 조회 (lock_sttus_code = 'LOCKED')

```

| tb_cm_account_lock | lock_sttus_code | 'LOCKED' → 'UNLOCK_PENDING' 상태 변경 |

#### FID-00021 잠금 해제 메일 재발송

#### 기능: [FID-00021] 잠금 해제 메일 재발송

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/unlock/email/resend` |
| **트리거** | [잠금 해제 메일 재발송] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| email | string | Y | 재발송할 이메일 주소 |

**처리 로직**
```
1. 서버에 재발송 API 호출
2. 서버 처리:
   a. tb_cm_account_lock에서 해당 회원의 활성 잠금 건 조회
   b. 새 해제 토큰 생성 → unlock_token_val UPDATE
   c. lock_sttus_code가 'LOCKED'이면 → 'UNLOCK_PENDING' 변경
   d. 해제 링크 포함 메일 재발송
3. 처리 중 버튼 비활성화
4. 성공 시 '잠금 해제 메일을 재발송했습니다' 토스트 표시
5. 60초 쿨타임 시작
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_account_lock | unlock_token_val | 새 해제 토큰으로 갱신 |
| tb_cm_account_lock | lock_sttus_code | 'UNLOCK_PENDING' 유지/변경 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 잠금 건 없음 | 404 | 잠금된 계정이 아닙니다 |
| 서버 오류 | 500 | 재발송 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |
