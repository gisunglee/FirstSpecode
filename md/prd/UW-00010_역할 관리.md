# UW-00010 역할 관리

> ## 1. 개요
| 항목 | 내용 |
|:-----|:-----|
| **단위업무ID** | UW-00010 |
| **단위업무명** | 역할 관리 |
| **비즈니스 목적** | OWNER/ADMIN이 멤버 목록에서 역할을 인라인으로 변경한다. OWNER는 반드시 1명 이상 유지되며, OWNER 승격은 OWNER 본인만 가능하다. |

#### FID-00072 멤버 목록 조회

#### 기능: [FID-00072] 멤버 목록 조회

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | SELECT |
| **API** | `GET /api/projects/{projectId}/members` |
| **트리거** | 화면 진입 시 자동 실행 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| members | array | 멤버 목록 (memberId, name, email, role, lastAccessedAt, hasWork) |
| totalCount | number | 전체 멤버 수 |
| myRole | string | 현재 사용자의 역할 (드롭다운 옵션 결정용) |
| ownerCount | number | 현재 OWNER 수 (마지막 OWNER 비활성화 판별용) |

**처리 로직**
```
1. 프로젝트 멤버 목록 조회 API 호출
2. 서버 처리:
   a. tb_pj_project_member에서 prjct_id = 현재 프로젝트 AND mber_sttus_code = 'ACTIVE' 건 조회
   b. tb_cm_member JOIN → mber_nm, email_addr 조회
   c. tb_pj_project_member.last_acces_dt → 마지막 접속일
   d. tb_pj_project_member에서 role_code = 'OWNER' 건수 카운트 → ownerCount
   e. 작업 현황 확인:
      - tb_ds_screen에서 asign_mber_id = 해당 멤버 건 존재 여부
      - tb_ds_area는 asign_mber_id 없음 → 제외
      - tb_ds_function에서 asign_mber_id = 해당 멤버 건 존재 여부
      - 하나라도 있으면 hasWork = true
3. 역할 드롭다운 옵션 결정
   - myRole = 'OWNER' → OWNER 옵션 포함
   - myRole = 'ADMIN' → OWNER 옵션 미포함
4. ownerCount = 1 이고 해당 멤버가 OWNER → 드롭다운 비활성화
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_pj_project_member | prjct_id | 프로젝트 필터 |
| tb_pj_project_member | mber_id | 멤버 FK |
| tb_pj_project_member | role_code | 역할 표시 + OWNER 카운트 |
| tb_pj_project_member | mber_sttus_code | 'ACTIVE' 필터 |
| tb_pj_project_member | last_acces_dt | 마지막 접속일 |
| tb_cm_member | mber_nm | 멤버 이름 |
| tb_cm_member | email_addr | 멤버 이메일 |
| tb_ds_screen | asign_mber_id | 화면 담당 배정 여부 확인 |
| tb_ds_function | asign_mber_id | 기능 담당 배정 여부 확인 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|

#### FID-00073 역할 변경

#### 기능: [FID-00073] 역할 변경

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `PATCH /api/projects/{projectId}/members/{memberId}/role` |
| **트리거** | 역할 드롭다운에서 항목 선택 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| role | string | Y | 변경할 역할 (OWNER/ADMIN/PM/DESIGNER/DEVELOPER/VIEWER) |

**처리 로직**
```
1. 드롭다운 선택 즉시 API 호출
2. 서버 처리:
   a. 요청자 권한 확인: tb_pj_project_member에서 요청자의 role_code 조회
      - ADMIN/OWNER만 변경 가능, 그 외 403
   b. OWNER 승격 검증:
      - role = 'OWNER' 요청 시 → 요청자 role_code = 'OWNER' 확인, 아니면 403
   c. 마지막 OWNER 보호:
      - 대상 멤버가 OWNER이고 role_code = 'OWNER' 건이 1건뿐이면 → 400 거부
   d. tb_pj_project_member UPDATE (role_code = 새 역할, sttus_chg_dt = NOW())
3. 성공 시 해당 행 역할 인라인 갱신
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_pj_project_member | role_code | 요청자 권한 확인 + 대상 역할 변경 + OWNER 카운트 |
| tb_pj_project_member | sttus_chg_dt | 변경 일시 기록 (NOW()) |
| tb_pj_project_member | mber_sttus_code | 'ACTIVE' 멤버만 대상 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 마지막 OWNER 변경 시도 | 400 | OWNER는 최소 1명 이상 유지되어야 합니다 |
| OWNER 승격 권한 없음 | 403 | OWNER 승격은 OWNER만 가능합니다 |
| 권한 없음 | 403 | 역할 변경 권한이 없습니다 |
| 서버 오류 | 500 | 역할 변경 중 오류가 발생했습니다 |
