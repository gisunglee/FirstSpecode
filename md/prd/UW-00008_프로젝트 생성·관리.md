#### FID-00056 프로젝트 생성

#### 기능: [FID-00056] 프로젝트 생성

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/projects` |
| **트리거** | [생성] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| name | string | Y | 프로젝트명 |
| description | string | N | 프로젝트 설명 |
| startDate | date | N | 시작일 |
| endDate | date | N | 종료일 |
| clientName | string | N | 고객사명 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| projectId | string | 생성된 프로젝트 ID |

**처리 로직**
```
1. 프로젝트명 공백 검증
2. 종료일이 시작일 이전인지 검증
3. 서버에 생성 API 호출
4. 서버 처리:
   a. tb_pj_project INSERT:
      - prjct_id: UUID 자동 생성
      - prjct_nm, prjct_dc, bgng_de, end_de, client_nm
      - creat_mber_id: 현재 로그인 회원
   b. tb_pj_project_member INSERT (생성자 자동 OWNER):
      - prjct_mber_id: UUID 자동 생성
      - prjct_id: 생성된 프로젝트
      - mber_id: 현재 로그인 회원
      - role_code: 'OWNER'
      - mber_sttus_code: 'ACTIVE'
   c. tb_pj_project_settings INSERT (기본 설정):
      - seting_id: UUID 자동 생성
      - prjct_id: 생성된 프로젝트
      - ai_call_mthd_code: 'DIRECT'
      - plan_code: 'FREE'
5. 성공 시 POPUP 닫기 + 목록 갱신
```

| tb_pj_project | prjct_dc | 설명 |
| tb_pj_project | bgng_de | 시작일 |
| tb_pj_project | end_de | 종료일 |
| tb_pj_project | client_nm | 고객사명 |
| tb_pj_project | creat_mber_id | 생성자 FK |

| tb_pj_project_member | mber_id | 생성자 FK |

```
1. 복사 API 호출 (처리 중 버튼 비활성화 및 로딩 표시)
2. 서버 처리:
   a. tb_pj_project INSERT (프로젝트명 = '원본명 (복사본)', creat_mber_id = 현재 회원)
   b. tb_pj_project_member INSERT (복사자 OWNER, 원본 멤버 미복사)

   d. 복사 대상 (원본 구조 전체 복사):
      - tb_rq_task (과업)
      - tb_rq_requirement (요구사항)
      - tb_rq_user_story (사용자스토리)
      - tb_rq_acceptance_criteria (인수기준)

      - tb_ds_area (영역)
      - tb_ds_function (기능)

      - tb_ds_table_column (테이블 컬럼)
      - tb_ds_function_column_mapping (기능 컬럼 매핑)
   e. 복사 제외 (이력·상태성 데이터):

      - tb_pj_project_invitation (초대)
      - tb_pj_project_api_key (API 키)
      - tb_rq_requirement_history (요구사항 이력)
      - tb_rq_baseline_snapshot (기준선 스냅샷)
      - tb_ds_table_column_history (컬럼 이력)
      - tb_ds_design_change (설계 변경 이력)

      - tb_cm_attach_file (첨부파일)

```

**처리 로직**
```
1. 삭제 API 호출 (처리 중 버튼 비활성화)

   a. OWNER 권한 확인
   b. 참여 멤버에게 제거 안내:
      - tb_pj_project_member에서 해당 프로젝트의 ACTIVE 멤버 목록 조회 (본인 제외)

      - tb_pj_project_member
      - tb_pj_project_invitation
      - tb_pj_project_api_key
      - tb_pj_project_settings
      - tb_pj_settings_history
      - tb_rq_task → tb_rq_requirement → tb_rq_user_story → tb_rq_acceptance_criteria
      - tb_rq_requirement_history, tb_rq_baseline_snapshot
      - tb_ds_unit_work → tb_ds_screen → tb_ds_area → tb_ds_function
      - tb_ds_db_table → tb_ds_table_column
      - tb_ds_function_column_mapping
      - tb_ds_table_column_history, tb_ds_design_change
      - tb_ai_task
      - tb_cm_attach_file (해당 prjct_id 건)
```
