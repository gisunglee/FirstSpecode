#### FID-00065 초대 발송

#### 기능: [FID-00065] 초대 발송

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/projects/{projectId}/invitations` |
| **트리거** | [초대 발송] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| invitations | array | Y | [{email, role}] 배열 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| results | array | 건별 성공/실패 여부 |

**처리 로직**
```
1. 모든 행 이메일 형식 검증
2. 역할 선택 여부 검증
3. 초대 발송 API 호출
4. 서버 처리 (건별):
   a. tb_pj_project_member에서 해당 이메일 회원이 이미 멤버인지 확인 → 409
   b. tb_pj_project_invitation에서 동일 이메일 PENDING 건 존재 여부 확인 → 409
   c. tb_pj_project_invitation INSERT:
      - invt_id: UUID 자동 생성
      - prjct_id: 현재 프로젝트
      - email_addr: 초대 이메일
      - role_code: 선택한 역할
      - invt_token_val: 랜덤 토큰 생성
      - invtr_mber_id: 현재 로그인 회원
      - invt_sttus_code: 'PENDING'
      - expiry_dt: NOW() + 7일
   d. 초대 메일 발송 (토큰 링크 포함)
5. 이미 속한 멤버/이미 초대 중 이메일은 해당 행만 인라인 에러 표시
6. 전체 성공 시 POPUP 닫기 + 초대 현황 갱신
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_pj_project_member | mber_id | 이미 멤버인지 확인 |
| tb_pj_project_member | mber_sttus_code | 'ACTIVE' 멤버 필터 |
| tb_pj_project_invitation | email_addr | 동일 이메일 PENDING 건 확인 |
| tb_pj_project_invitation | invt_sttus_code | 'PENDING' 중복 확인 |
| tb_pj_project_invitation | invt_id | PK, UUID 자동 생성 |
| tb_pj_project_invitation | prjct_id | 현재 프로젝트 FK |
| tb_pj_project_invitation | role_code | 초대 역할 |
| tb_pj_project_invitation | invt_token_val | 초대 토큰 값 |
| tb_pj_project_invitation | invtr_mber_id | 초대자 회원 FK |
| tb_pj_project_invitation | expiry_dt | NOW() + 7일 |
| tb_cm_member | email_addr | 이메일로 기존 회원 조회 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 이메일 형식 불일치 | - | 올바른 이메일 형식을 입력해 주세요 |
| 이미 속한 멤버 | 409 | 이미 프로젝트에 속한 멤버입니다 |
| 이미 초대 중인 이메일 | 409 | 이미 초대 중인 이메일입니다 |
| 서버 오류 | 500 | 초대 발송 중 오류가 발생했습니다 |

```
   b. invt_sttus_code = 'PENDING' AND expiry_dt > NOW() 확인
   c. tb_pj_project JOIN으로 프로젝트명 조회
   d. tb_cm_member JOIN으로 초대자 이메일 조회 (invtr_mber_id)
3. 유효 시 초대 정보 표시
4. 로그인 상태 확인 → 로그인이면 수락하기 버튼 표시, 미로그인이면 로그인/회원가입 링크 표시
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_pj_project_invitation | invt_token_val | 토큰 조회 |
| tb_pj_project_invitation | invt_sttus_code | 'PENDING' 확인 |
| tb_pj_project_invitation | expiry_dt | 만료 여부 확인 |
| tb_pj_project_invitation | role_code | 부여 역할 표시 |
| tb_pj_project_invitation | invtr_mber_id | 초대자 FK |
| tb_pj_project | prjct_nm | 프로젝트명 표시 |
| tb_cm_member | email_addr | 초대자 이메일 표시 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 토큰 만료 | 400 | 초대 링크가 만료되었습니다. 초대 재발송을 요청해 주세요 |
| 토큰 무효 | 400 | 유효하지 않은 초대 링크입니다 |

#### FID-00070 초대 수락 처리

#### 기능: [FID-00070] 초대 수락 처리

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `POST /api/invitations/{inviteToken}/accept` |
| **트리거** | [수락하기] 버튼 클릭 시 (로그인 상태) |

**처리 로직**
```
1. 수락 API 호출
2. 서버 처리:
   a. tb_pj_project_invitation에서 invt_token_val 재검증 (PENDING + 미만료)
   b. tb_pj_project_invitation UPDATE (invt_sttus_code = 'ACCEPTED', accept_dt = NOW())
   c. tb_pj_project_member INSERT:
      - prjct_mber_id: UUID 자동 생성
      - prjct_id: 초대 프로젝트
      - mber_id: 현재 로그인 회원
      - role_code: 초대 시 지정한 역할
      - mber_sttus_code: 'ACTIVE'
3. 성공 시 해당 프로젝트 메인 화면으로 이동
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_pj_project_invitation | invt_token_val | 토큰 재검증 |
| tb_pj_project_invitation | invt_sttus_code | 'PENDING' → 'ACCEPTED' 상태 변경 |
| tb_pj_project_invitation | accept_dt | 수락 일시 기록 (NOW()) |
| tb_pj_project_invitation | role_code | 멤버 등록 시 역할 참조 |
| tb_pj_project_member | prjct_mber_id | PK, UUID 자동 생성 |
| tb_pj_project_member | prjct_id | 프로젝트 FK |
| tb_pj_project_member | mber_id | 수락 회원 FK |
| tb_pj_project_member | role_code | 초대 시 지정 역할 |
| tb_pj_project_member | mber_sttus_code | 'ACTIVE' |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 토큰 만료 | 400 | 초대 링크가 만료되었습니다 |
| 이미 멤버 | 409 | 이미 해당 프로젝트의 멤버입니다 |
| 서버 오류 | 500 | 수락 처리 중 오류가 발생했습니다 |
