# UW-00007 회원 탈퇴

> ## 1. 개요
| 항목 | 내용 |
|:-----|:-----|
| **단위업무ID** | UW-00007 |
| **단위업무명** | 회원 탈퇴 |
| **비즈니스 목적** | 사용자가 직접 계정을 삭제한다. 소유 프로젝트 보유 여부에 따라 처리 흐름이 분기되며, 재인증 또는 확인 다이얼로그를 통해 실수 탈퇴를 방지한다. 탈퇴 시 회원 상태를 WITHDRAWN으로 변경(논리 삭제)하고, 양도되지 않은 소유 프로젝트는 물리 삭제한다. 참여 프로젝트 멤버에게는 제거 안내를 남긴다. 동일 이메일로 재가입이 가능하다. |
| **관련 요구사항** | RQ-00007, RQ-00001, RQ-00004 |

## 2. 화면 목록
| 화면ID | 화면명 | URL | 유형 | 설명 |
|:-------|:-------|:----|:-----|:-----|
| PID-00014 | 회원 탈퇴 | /settings/account/withdraw | DETAIL | 소유 프로젝트 처리 → 재인증 → 탈퇴 완료 단일 화면 스텝 전환 |

## 3. 화면 흐름
```
[설정 > 계정]
    │ [회원 탈퇴] 버튼 클릭
    ▼
[회원 탈퇴 화면]
    │
    ├─ 소유 프로젝트 있음
    │       │ STEP 1: 소유 프로젝트 확인
    │       │  ├─ [양도하기] → 프로젝트 설정 화면으로 이동 → 양도 후 복귀
    │       │  └─ [삭제 후 탈퇴] → STEP 2
    │       │ STEP 2: 재인증
    │       │  ├─ 이메일 계정: 비밀번호 입력
    │       │  └─ 소셜 계정: OAuth 팝업
    │       ▼
    │     탈퇴 완료 → 논리 삭제 + 전 세션 무효화 → 로그인 화면
    │
    └─ 소유 프로젝트 없음
            │ 확인 다이얼로그
            ▼
          탈퇴 완료 → 논리 삭제 + 전 세션 무효화 → 로그인 화면
```

| 이동 | 전달 파라미터 | 동작 |
|:-----|:-------------|:-----|
| 설정 → 탈퇴 화면 | 없음 | 회원 탈퇴 버튼 클릭 |
| 탈퇴 화면 → 프로젝트 설정 | projectId | 양도하기 클릭 |
| 프로젝트 설정 → 탈퇴 화면 | 없음 | 양도 완료 후 복귀 |
| 탈퇴 완료 → 로그인 | 없음 | 논리 삭제 + 전 세션 무효화 후 이동 |

| 항목 | 내용 |
|:-----|:-----|

| **트리거** | 화면 진입 시 자동 실행, 프로젝트 설정에서 복귀 시 재조회 |

```

#### FID-00050 비밀번호 재인증 후 탈퇴

#### 기능: [FID-00050] 비밀번호 재인증 후 탈퇴

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | DELETE |
| **API** | `DELETE /api/member/me` |
| **트리거** | [탈퇴 확인] 버튼 클릭 (이메일 계정) |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| password | string | Y | 현재 비밀번호 |

**처리 로직**
```
1. 비밀번호 미입력 시 API 호출 안 함
2. API 호출 → 버튼 로딩 상태 전환
3. 서버 처리:
   a. tb_cm_member에서 pswd_hash와 bcrypt 비교 → 불일치 시 401
   b. 회원 논리 삭제:
      - tb_cm_member UPDATE (mber_sttus_code = 'WITHDRAWN', wthdrw_dt = NOW())
   c. 소유 프로젝트 물리 삭제:
      - tb_pj_project DELETE WHERE creat_mber_id = 해당 회원 (CASCADE로 하위 데이터 전체 삭제)
   d. 참여 프로젝트 멤버 제거:
      - tb_pj_project_member UPDATE (mber_sttus_code = 'LEFT', sttus_chg_dt = NOW()) WHERE mber_id = 해당 회원
   e. 참여 프로젝트 멤버 제거 안내:
      - tb_pj_member_removal_notice INSERT (각 참여 프로젝트의 OWNER에게, prjct_nm 포함)
   f. 소셜 계정 연동 해제:
      - tb_cm_social_account DELETE WHERE mber_id = 해당 회원
   g. 전 세션·RT 무효화:
      - tb_cm_refresh_token: 모든 활성 RT → revoke_dt = NOW()
      - tb_cm_member_session: 모든 활성 세션 → invald_dt = NOW()
4. 완료 → 로그인 화면으로 이동
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | pswd_hash | 비밀번호 bcrypt 비교 |
| tb_cm_member | mber_sttus_code | 'WITHDRAWN' 상태 변경 |
| tb_cm_member | wthdrw_dt | 탈퇴 일시 기록 (NOW()) |
| tb_pj_project | prjct_id | 소유 프로젝트 물리 삭제 (CASCADE) |
| tb_pj_project | creat_mber_id | 소유자 판별 |
| tb_pj_project_member | mber_sttus_code | 'LEFT' 상태 변경 (참여 프로젝트) |
| tb_pj_project_member | sttus_chg_dt | 상태 변경 일시 |
| tb_pj_member_removal_notice | notice_id | PK, UUID 자동 생성 |
| tb_pj_member_removal_notice | mber_id | 안내 대상 (프로젝트 OWNER) |
| tb_pj_member_removal_notice | prjct_id | 참여했던 프로젝트 |
| tb_pj_member_removal_notice | prjct_nm | 프로젝트명 스냅샷 |
| tb_cm_social_account | mber_id | 소셜 계정 삭제 대상 |
| tb_cm_refresh_token | revoke_dt | 모든 활성 RT → NOW() |
| tb_cm_member_session | invald_dt | 모든 활성 세션 → NOW() |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 비밀번호 불일치 | 401 | 비밀번호가 올바르지 않습니다 |
| 서버 오류 | 500 | 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |

3. 완료 → 로그인 화면으로 이동
```
