# UW-00003 소셜 로그인

> ## 1. 개요
| 항목 | 내용 |
|:-----|:-----|
| **단위업무ID** | UW-00003 |
| **단위업무명** | 소셜 로그인 |
| **비즈니스 목적** | Google·GitHub OAuth 인증으로 간편 로그인·가입을 처리한다. 최초 로그인 시 계정 자동 생성, 기존 이메일 계정과 동일한 이메일 감지 시 사용자 확인 후 연동 처리한다. |
| **관련 요구사항** | RQ-00003, RQ-00002, RQ-00004 |
| **기술 스택** | Google OAuth 2.0, GitHub OAuth Apps |

## 2. 화면 목록
| 화면ID | 화면명 | URL | 유형 | 설명 |
|:-------|:-------|:----|:-----|:-----|
| [신규] | 소셜 로그인 콜백 | /auth/social/callback | DETAIL | OAuth 인증 성공 후 콜백 처리 (신규가입/기존로그인/연동분기) |
| PID-00008 | 소셜 계정 연동 확인 | /auth/social/link | POPUP | 동일 이메일 감지 시 기존 계정 연동 여부 확인 |

## 3. 화면 흐름
```
[로그인 화면] ──(Google/GitHub 버튼 클릭)──▶ [OAuth 인증 페이지 (외부)]
[OAuth 인증 페이지] ──(인증 성공)──▶ [소셜 로그인 콜백]
[소셜 로그인 콜백] ──(신규·기존 소셜)──▶ [대시보드]
[소셜 로그인 콜백] ──(동일 이메일 감지)──▶ [PID-00008 소셜 계정 연동 확인 POPUP]
[PID-00008 연동 확인] ──(확인)──▶ [대시보드]
[PID-00008 연동 확인] ──(취소)──▶ [로그인 화면]
```

| 이동 | 전달 파라미터 | 동작 |
|:-----|:-------------|:-----|
| 로그인 화면 → OAuth | provider (google/github) | 소셜 로그인 버튼 클릭 |
| OAuth → 콜백 화면 | code, state (URL 파라미터) | OAuth 인증 성공 후 리다이렉트 |
| 콜백 → 대시보드 | - | 신규 가입 또는 기존 소셜 로그인 완료 |
| 콜백 → 연동 확인 POPUP | email, socialToken | 동일 이메일 감지 시 표시 |
| 연동 확인 POPUP → 대시보드 | - | 연동 확인 후 로그인 완료 |
| 연동 확인 POPUP → 로그인 화면 | - | 취소 시 복귀 |

#### FID-00200 OAuth 콜백 처리

#### 기능: [신규] OAuth 콜백 처리

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/social/callback` |
| **트리거** | 화면 진입 시 자동 실행 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| code | string | Y | OAuth 인증 코드 (URL 파라미터) |
| state | string | Y | CSRF 방지 state 값 (URL 파라미터) |
| provider | string | Y | 'google' 또는 'github' (state에서 추출) |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| resultType | string | NEW(신규가입) / EXISTING(기존로그인) / LINK_REQUIRED(연동필요) |
| accessToken | string | JWT AT (NEW/EXISTING 시) |
| refreshToken | string | JWT RT (NEW/EXISTING 시) |
| socialToken | string | 임시 소셜 토큰 (LINK_REQUIRED 시) |
| email | string | 감지된 이메일 (LINK_REQUIRED 시) |

**처리 로직**
```
1. URL에서 code, state 파라미터 추출
2. 서버에 콜백 API 호출
3. 서버 처리:
   a. state 검증 (세션에 저장된 값과 비교)
   b. OAuth Provider에 code → Access Token 교환
   c. Provider에서 사용자 정보(email, sub/id) 조회
   d. tb_cm_social_account에서 provdr_code + provdr_user_id로 조회
      → 존재: 기존 소셜 로그인 (resultType = 'EXISTING')
   e. 미존재 시 tb_cm_member에서 email_addr로 조회
      → 존재: 동일 이메일 감지 (resultType = 'LINK_REQUIRED', socialToken 발급)
      → 미존재: 신규 가입 (resultType = 'NEW')
   f. NEW 시:
      - tb_cm_member INSERT (mber_sttus_code = 'ACTIVE', pswd_hash = NULL)
      - tb_cm_social_account INSERT
      - tb_cm_refresh_token INSERT (expiry_dt = NOW() + 10일)
      - tb_cm_member_session INSERT
      - AT/RT 발급
   g. EXISTING 시:
      - tb_cm_refresh_token INSERT
      - tb_cm_member_session INSERT
      - AT/RT 발급
4. 클라이언트:
   a. NEW/EXISTING → 토큰 저장 → 대시보드 이동
   b. LINK_REQUIRED → socialToken, email 저장 → PID-00008 팝업 표시
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | mber_id | PK, UUID 자동 생성 (신규 시) |
| tb_cm_member | email_addr | 동일 이메일 조회 |
| tb_cm_member | mber_sttus_code | 'ACTIVE' (소셜은 인증 불요) |
| tb_cm_social_account | provdr_code | 'GOOGLE' 또는 'GITHUB' |
| tb_cm_social_account | provdr_user_id | OAuth sub/id |
| tb_cm_social_account | provdr_email_addr | Provider 이메일 |
| tb_cm_refresh_token | token_hash_val | RT 해시 저장 |
| tb_cm_refresh_token | expiry_dt | NOW() + 10일 |
| tb_cm_member_session | device_info_cn | User-Agent 등 |
| tb_cm_member_session | ip_addr | 클라이언트 IP |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| state 불일치 (CSRF) | 400 | 인증 요청이 유효하지 않습니다. 다시 시도해 주세요 |
| code 교환 실패 | 400 | 소셜 인증에 실패했습니다. 다시 시도해 주세요 |
| Provider 사용자 정보 조회 실패 | 502 | 소셜 서비스 연결에 실패했습니다. 잠시 후 다시 시도해 주세요 |
| 서버 오류 | 500 | 로그인 처리 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |

#### FID-00024 소셜 계정 연동 처리

#### 기능: [FID-00024] 소셜 계정 연동 처리

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `POST /api/auth/social/link` |
| **트리거** | [연동하기] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| socialToken | string | Y | OAuth 콜백에서 전달된 임시 소셜 인증 토큰 |
| email | string | Y | 연동할 기존 계정 이메일 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| accessToken | string | JWT Access Token |
| refreshToken | string | JWT Refresh Token |

**처리 로직**
```
1. 서버에 소셜 계정 연동 API 호출
2. 서버 처리:
   a. socialToken 검증 → Provider 정보(provdr_code, provdr_user_id, provdr_email_addr) 추출
   b. tb_cm_member에서 email_addr로 기존 회원 조회
   c. tb_cm_social_account INSERT (기존 회원의 mber_id에 소셜 계정 연결)
   d. tb_cm_refresh_token INSERT (expiry_dt = NOW() + 10일)
   e. tb_cm_member_session INSERT
   f. AT/RT 발급
3. 처리 중 버튼 비활성화
4. 성공 시 토큰 저장 후 대시보드로 이동
```

| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | email_addr | 기존 회원 조회 |
| tb_cm_member | mber_id | 연동 대상 회원 PK |
| tb_cm_social_account | social_acnt_id | PK, UUID 자동 생성 |
| tb_cm_social_account | mber_id | 기존 회원 FK |
| tb_cm_social_account | provdr_code | 'GOOGLE' 또는 'GITHUB' |
| tb_cm_social_account | provdr_user_id | OAuth sub/id |
| tb_cm_social_account | provdr_email_addr | Provider 이메일 |
| tb_cm_refresh_token | token_id | PK, UUID 자동 생성 |
| tb_cm_refresh_token | mber_id | 회원 FK |
| tb_cm_refresh_token | token_hash_val | RT 해시 저장 |
