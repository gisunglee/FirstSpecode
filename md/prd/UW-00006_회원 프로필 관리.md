#### FID-00040 이메일 변경 요청

#### 기능: [FID-00040] 이메일 변경 요청

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/member/profile/email/change` |
| **트리거** | [변경 요청] 버튼 클릭 후 새 이메일 입력 및 확인 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| newEmail | string | Y | 변경할 새 이메일 주소 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| newEmail | string | 인증 메일 발송된 이메일 (PID-00013 전달용) |

**처리 로직**
```
1. 새 이메일 형식 검증
2. 서버에 변경 요청 API 호출
3. 서버 처리:
   a. tb_cm_member에서 email_addr로 새 이메일 중복 확인 (UK)
   b. tb_cm_email_verification에서 해당 회원의 기존 EMAIL_CHANGE PENDING 건 → vrfctn_sttus_code = 'EXPIRED' 처리
   c. tb_cm_email_verification INSERT (vrfctn_ty_code = 'EMAIL_CHANGE', email_addr = newEmail, vrfctn_sttus_code = 'PENDING', expiry_dt = NOW() + 1시간)
   d. 새 이메일로 인증 메일 발송 (토큰 링크 포함)
4. 성공 시 이메일 변경 인증 안내 화면(PID-00013)으로 이동 (newEmail 전달)
```

#### FID-00041 비밀번호 변경

#### 기능: [FID-00041] 비밀번호 변경

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `PUT /api/member/profile/password` |
| **트리거** | [비밀번호 변경] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| currentPassword | string | N | 현재 비밀번호 (소셜 전용 계정은 불필요) |
| newPassword | string | Y | 새 비밀번호 |

**처리 로직**
```
1. 현재 비밀번호 확인 (소셜 전용 계정은 생략)
2. 새 비밀번호 복잡도 검증 (영문+숫자+특수문자 8자 이상)
3. 새 비밀번호·확인 일치 여부 검증
4. 서버에 비밀번호 변경 API 호출
5. 서버 처리:
   a. pswd_hash IS NOT NULL이면 currentPassword bcrypt 비교
   b. tb_cm_member UPDATE (pswd_hash = 새 비밀번호 bcrypt 해시, mdfcn_dt = NOW())
   c. 현재 세션을 제외한 나머지 세션 무효화:
      - tb_cm_refresh_token: 현재 RT 제외 나머지 활성 RT → revoke_dt = NOW()
      - tb_cm_member_session: 현재 세션 제외 나머지 활성 세션 → invald_dt = NOW()
6. 성공 시 '비밀번호가 변경되었습니다' 토스트 표시
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | pswd_hash | 현재 비밀번호 비교 + 새 bcrypt 해시 저장 |
| tb_cm_member | mdfcn_dt | 수정 일시 갱신 |
| tb_cm_refresh_token | revoke_dt | 현재 RT 제외 나머지 → NOW() |
| tb_cm_member_session | invald_dt | 현재 세션 제외 나머지 → NOW() |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 현재 비밀번호 불일치 | 401 | 현재 비밀번호가 올바르지 않습니다 |
| 복잡도 미충족 | - | 비밀번호는 영문·숫자·특수문자를 포함한 8자 이상이어야 합니다 |
| 새 비밀번호 불일치 | - | 비밀번호가 일치하지 않습니다 |
| 서버 오류 | 500 | 비밀번호 변경 중 오류가 발생했습니다 |

#### FID-00042 소셜 계정 연동 추가

#### 기능: [FID-00042] 소셜 계정 연동 추가

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/member/social/link` |
| **트리거** | [연동하기] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| provider | string | Y | google / github |
| socialToken | string | Y | OAuth 인증 후 발급된 토큰 |

**처리 로직**
```
1. OAuth 인증 페이지로 이동
2. 인증 성공 후 콜백으로 소셜 토큰 수신
3. 연동 API 호출
4. 서버 처리:
   a. socialToken 검증 → Provider 정보 추출
   b. tb_cm_social_account INSERT (mber_id = 현재 회원, provdr_code, provdr_user_id, provdr_email_addr)
5. 성공 시 '연동이 완료되었습니다' 토스트 표시 및 목록 갱신
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_social_account | social_acnt_id | PK, UUID 자동 생성 |
| tb_cm_social_account | mber_id | 현재 회원 FK |
| tb_cm_social_account | provdr_code | 'GOOGLE' 또는 'GITHUB' |
| tb_cm_social_account | provdr_user_id | OAuth sub/id |
| tb_cm_social_account | provdr_email_addr | Provider 이메일 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 이미 연동된 계정 | 409 | 이미 연동된 소셜 계정입니다 |
| 서버 오류 | 500 | 연동 중 오류가 발생했습니다 |

#### FID-00043 소셜 계정 연동 해제

#### 기능: [FID-00043] 소셜 계정 연동 해제

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | DELETE |
| **API** | `DELETE /api/member/social/unlink` |
| **트리거** | [해제] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| provider | string | Y | google / github |

**처리 로직**
```
1. 서버에 연동 해제 API 호출
2. 서버 처리:
   a. 마지막 로그인 수단 여부 확인 (pswd_hash IS NULL AND 소셜 계정 1개면 해제 불가)
   b. tb_cm_social_account DELETE (해당 provdr_code 건)
3. 성공 시 '연동이 해제되었습니다' 토스트 표시 및 목록 갱신
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|

| tb_cm_social_account | provdr_code | 해제 대상 Provider 조회 |
| tb_cm_social_account | social_acnt_id | 삭제 대상 PK |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 마지막 로그인 수단 | 400 | 마지막 로그인 수단은 해제할 수 없습니다 |
| 서버 오류 | 500 | 연동 해제 중 오류가 발생했습니다 |

#### FID-00045 이메일 변경 인증 메일 재발송

#### 기능: [FID-00045] 이메일 변경 인증 메일 재발송

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/member/profile/email/resend` |
| **트리거** | [재발송] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| newEmail | string | Y | 재발송할 새 이메일 주소 |

**처리 로직**
```
1. 서버에 재발송 API 호출
2. 서버 처리:
   a. tb_cm_email_verification에서 기존 EMAIL_CHANGE PENDING 건 → vrfctn_sttus_code = 'EXPIRED'
   b. tb_cm_email_verification INSERT (새 토큰, vrfctn_ty_code = 'EMAIL_CHANGE', vrfctn_sttus_code = 'PENDING', expiry_dt = NOW() + 1시간)
   c. 인증 메일 재발송
3. 처리 중 버튼 비활성화
4. 성공 시 '인증 메일을 재발송했습니다' 토스트 표시
5. 60초 쿨타임 시작
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_email_verification | vrfctn_sttus_code | 기존 PENDING → 'EXPIRED' |
| tb_cm_email_verification | vrfctn_id | 새 인증 레코드 PK |
| tb_cm_email_verification | email_addr | 새 이메일 주소 |
| tb_cm_email_verification | vrfctn_token_val | 새 인증 토큰 |
| tb_cm_email_verification | vrfctn_ty_code | 'EMAIL_CHANGE' |
| tb_cm_email_verification | expiry_dt | NOW() + 1시간 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 서버 오류 | 500 | 재발송 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |
