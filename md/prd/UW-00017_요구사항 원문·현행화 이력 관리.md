```
2. 확정 API 호출
3. 서버 처리:
   a. tb_rq_requirement에서 prjct_id = 현재 프로젝트 전체 조회
   b. 전체 요구사항의 orgnl_cn, curncy_cn, spec_cn 등을 JSONB로 직렬화
   c. tb_rq_baseline_snapshot INSERT:
      - basln_id: UUID 자동 생성
      - basln_nm: 입력한 기준선명
      - coment_cn: 확정 코멘트
      - req_cnt: 현재 요구사항 수
      - snapshot_data: 전체 요구사항 JSONB 스냅샷
      - cnfrm_mber_id: 현재 회원
4. 성공 시 '기준선이 저장되었습니다' 토스트 표시 + 목록 갱신
```
