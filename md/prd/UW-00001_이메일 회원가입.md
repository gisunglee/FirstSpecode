#### FID-00005 회원가입 버튼 — 인증 메일 발송 요청

#### 기능: [FID-00005] 회원가입 버튼 — 인증 메일 발송 요청

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/register` |
| **트리거** | [회원가입] 버튼 클릭 시 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| email | string | Y | 가입 이메일 주소 |
| password | string | Y | 검증 통과한 비밀번호 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| email | string | 인증 메일이 발송된 이메일 주소 (PID-00004 전달용) |

**처리 로직**
```
1. 전체 필드 일괄 유효성 재검증
2. 통과 시 서버에 가입 요청 API 호출
3. 서버 처리:
   a. tb_cm_member INSERT (mber_sttus_code = 'UNVERIFIED', pswd_hash = bcrypt 해시)
   b. tb_cm_email_verification INSERT (vrfctn_ty_code = 'REGISTER', vrfctn_sttus_code = 'PENDING', expiry_dt = NOW() + 1시간)
   c. 인증 메일 발송 (토큰 링크 포함)
4. 처리 중 버튼 비활성화 및 로딩 표시
5. 성공 시 인증 메일 발송 안내 화면(PID-00004)으로 이동 (email 전달)
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | mber_id | PK, UUID 자동 생성 |
| tb_cm_member | email_addr | 가입 이메일 저장 |
| tb_cm_member | pswd_hash | 비밀번호 bcrypt 해시 저장 |
| tb_cm_member | mber_sttus_code | 'UNVERIFIED' 초기값 |
| tb_cm_member | join_dt | 가입 일시 (DEFAULT CURRENT_TIMESTAMP) |
| tb_cm_email_verification | vrfctn_id | PK, UUID 자동 생성 |
| tb_cm_email_verification | mber_id | 가입 회원 FK |
| tb_cm_email_verification | email_addr | 인증 대상 이메일 |
| tb_cm_email_verification | vrfctn_token_val | 인증 토큰 값 |
| tb_cm_email_verification | vrfctn_ty_code | 'REGISTER' |
| tb_cm_email_verification | vrfctn_sttus_code | 'PENDING' 초기값 |
| tb_cm_email_verification | expiry_dt | 만료 일시 (NOW() + 1시간) |

```
   a. tb_cm_email_verification에서 기존 PENDING 건 → vrfctn_sttus_code = 'EXPIRED' 처리
   b. tb_cm_email_verification INSERT (새 토큰, vrfctn_ty_code = 'REGISTER', vrfctn_sttus_code = 'PENDING', expiry_dt = NOW() + 1시간)
   c. 인증 메일 재발송
3. 처리 중 버튼 비활성화
4. 성공 시 '인증 메일을 재발송했습니다' 토스트 메시지 표시
5. 60초 쿨타임 시작
```

#### FID-00012 가입 완료 메시지 표시

#### 기능: [FID-00012] 가입 완료 메시지 표시

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `POST /api/auth/verify` |
| **트리거** | 화면 진입 시 자동 실행 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| token | string | Y | URL 파라미터의 인증 토큰 |

**Output**
| 필드 | 타입 | 설명 |
|:-----|:-----|:-----|
| accessToken | string | JWT 액세스 토큰 (자동 로그인용) |
| refreshToken | string | 리프레시 토큰 (자동 로그인용) |

**처리 로직**
```
1. URL 파라미터의 인증 토큰을 서버에 검증 요청
2. 서버 처리:
   a. tb_cm_email_verification에서 vrfctn_token_val 조회
   b. vrfctn_sttus_code = 'PENDING' 이고 expiry_dt > NOW() 인지 확인
   c. 유효 시:
      - tb_cm_email_verification UPDATE (vrfctn_sttus_code = 'VERIFIED', vrfctn_dt = NOW())
      - tb_cm_member UPDATE (mber_sttus_code = 'UNVERIFIED' → 'ACTIVE')
      - JWT 액세스 토큰 발급
      - tb_cm_refresh_token INSERT (자동 로그인용 리프레시 토큰 저장)
   d. 만료·무효 시: 에러 응답
3. 클라이언트:
   a. 성공 시 accessToken/refreshToken 저장 후 '회원가입이 완료되었습니다!' 메시지 표시
   b. 실패 시 에러 메시지 표시 및 재발송 안내 이동 버튼 제공
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_email_verification | vrfctn_token_val | 토큰 조회 |
| tb_cm_email_verification | vrfctn_sttus_code | 'PENDING' → 'VERIFIED' 상태 변경 |
| tb_cm_email_verification | expiry_dt | 만료 여부 확인 |
| tb_cm_email_verification | vrfctn_dt | 인증 일시 기록 (NOW()) |
| tb_cm_member | mber_sttus_code | 'UNVERIFIED' → 'ACTIVE' 상태 변경 |
| tb_cm_refresh_token | token_id | PK, UUID 자동 생성 |
| tb_cm_refresh_token | mber_id | 인증 완료 회원 FK |
| tb_cm_refresh_token | token_hash_val | 리프레시 토큰 해시 저장 |
| tb_cm_refresh_token | expiry_dt | 만료 일시 (기본 10일) |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 토큰 만료 | 400 | 인증 링크가 만료되었습니다 |
| 토큰 무효 | 400 | 유효하지 않은 인증 링크입니다 |
