# UW-00004 JWT 인증 처리

> ## 1. 개요
| 항목 | 내용 |
|:-----|:-----|
| **단위업무ID** | UW-00004 |
| **단위업무명** | JWT 인증 처리 |
| **비즈니스 목적** | 로그인 성공 시 Access Token(1시간)·Refresh Token(10일)을 발급하고 인증 상태를 유지한다. Refresh Token은 httpOnly 쿠키로 저장하며, Access Token 만료 시 자동 갱신한다. 다중 기기 로그인을 허용하며 로그아웃 시 해당 기기 토큰·세션만 무효화한다. |
| **관련 요구사항** | RQ-00004, RQ-00002, RQ-00003, RQ-00007 |
| **기술 스택** | JWT, httpOnly Cookie, SameSite 쿠키 설정 |

> ⚙️ **백엔드 처리 전용 단위업무** — 사용자가 직접 접근하는 UI 화면 없음. 전체 시스템 공통 인증 처리 로직으로 구성됨.

## 2. 화면 목록
> 해당 없음 (백엔드 처리 전용)

## 3. 화면 흐름
```
[로그인 성공] ──▶ AT(1시간) + RT(10일) 발급
[API 요청] ──(AT 유효)──▶ 정상 처리
[API 요청] ──(AT 만료)──▶ RT로 AT 자동 갱신 ──▶ 정상 처리
[API 요청] ──(RT 만료·무효)──▶ 401 반환 ──▶ 로그인 화면으로 리다이렉트
[로그아웃] ──▶ 해당 기기 RT 무효화 + 세션 무효화
```

| 처리 | 조건 | 동작 |
|:-----|:-----|:-----|
| AT 발급 | 로그인 성공 시 | 만료 1시간, 메모리(JavaScript 변수) 저장 |
| RT 발급 | 로그인 성공 시 | 만료 10일, httpOnly 쿠키 저장 |
| AT 자동 갱신 | AT 만료, RT 유효 시 | RT로 새 AT 발급 |
| RT 자동 연장 | 자동 로그인 선택 시 | RT 만료 전 자동 갱신으로 로그인 상태 유지 |
| 토큰 무효화 | 로그아웃 시 | 해당 기기 RT revoke_dt = NOW() 처리 |
| 세션 무효화 | 로그아웃 시 | 해당 기기 세션 invald_dt = NOW() 처리 |
| 401 반환 | AT·RT 모두 무효 시 | 클라이언트 AT 삭제 → 로그인 화면으로 리다이렉트 |
