# UW-00005 비밀번호 재설정

> ## 1. 개요
| 항목 | 내용 |
|:-----|:-----|
| **단위업무ID** | UW-00005 |
| **단위업무명** | 비밀번호 재설정 |
| **비즈니스 목적** | 비밀번호를 분실한 회원이 이메일 링크를 통해 안전하게 비밀번호를 재설정할 수 있도록 한다. 계정 존재 여부를 노출하지 않는 보안 처리와 전 세션 무효화를 통해 보안성을 확보한다. |
| **관련 요구사항** | RQ-00005, RQ-00002, RQ-00004, RQ-00003 |

#### FID-00026 재설정 링크 발송 요청

#### 기능: [FID-00026] 재설정 링크 발송 요청

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | INSERT |
| **API** | `POST /api/auth/password/reset-request` |
| **트리거** | [재설정 링크 발송] 버튼 클릭, 이메일 입력 후 엔터 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| email | string | Y | 재설정 링크를 받을 이메일 주소 |

**처리 로직**
```
1. 이메일 형식 클라이언트 검증 (형식 오류 시 API 호출 안 함)
2. API 호출 → 버튼 로딩 상태 전환
3. 서버 처리:
   a. tb_cm_member에서 email_addr로 회원 조회
   b. [미가입 이메일] → 정상 응답과 동일 처리 (보안 - 가입 여부 노출 차단)
   c. [소셜 전용 계정 (pswd_hash IS NULL)] → 소셜 계정 안내 응답
   d. [정상 계정]:
      - tb_cm_password_reset_token에서 해당 회원의 기존 PENDING 건 → token_sttus_code = 'EXPIRED' 처리
      - tb_cm_password_reset_token INSERT (token_val = 랜덤 토큰, token_sttus_code = 'PENDING', expiry_dt = NOW() + 1시간)
      - 재설정 링크 포함 이메일 발송
4. 응답 성공 시 → 발송 완료 안내 영역(STEP 2)으로 화면 상태 전환
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_member | email_addr | 회원 조회 |
| tb_cm_member | pswd_hash | 소셜 전용 계정 판별 (NULL이면 소셜 전용) |
| tb_cm_password_reset_token | reset_token_id | PK, UUID 자동 생성 |
| tb_cm_password_reset_token | mber_id | 대상 회원 FK |
| tb_cm_password_reset_token | token_val | 랜덤 토큰 값 |
| tb_cm_password_reset_token | token_sttus_code | 기존 PENDING → 'EXPIRED', 신규 'PENDING' |
| tb_cm_password_reset_token | expiry_dt | NOW() + 1시간 |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|
| 이메일 형식 오류 | - | 올바른 이메일 형식으로 입력해 주세요 (클라이언트) |
| 소셜 전용 계정 | 200 | 소셜 계정으로 가입된 계정입니다. Google/GitHub로 로그인해 주세요 |
| 서버 오류 | 500 | 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요 |

#### FID-00030 토큰 유효성 검증

#### 기능: [FID-00030] 토큰 유효성 검증

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | SELECT |
| **API** | `GET /api/auth/password/reset?token={token}` |
| **트리거** | 화면 진입 시 자동 실행 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| token | string | Y | URL query parameter |

**처리 로직**
```
1. URL에서 token 추출
2. 토큰 유효성 검증 API 호출
3. 서버 처리:
   a. tb_cm_password_reset_token에서 token_val 조회
   b. token_sttus_code = 'PENDING' 확인 → 아니면 USED/INVALID 분기
   c. expiry_dt > NOW() 확인 → 아니면 EXPIRED 분기
   d. 유효 시 200 응답
4. 클라이언트:
   a. 유효 → 새 비밀번호 입력 폼 표시
   b. 무효 → /auth/password/invalid?reason={reason}으로 리다이렉트
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_password_reset_token | token_val | 토큰 조회 |
| tb_cm_password_reset_token | token_sttus_code | 'PENDING' 여부 확인 |
| tb_cm_password_reset_token | expiry_dt | 만료 여부 확인 |

**에러 처리**
| 상황 | HTTP | 처리 |
|:-----|:-----|:-----|
| 토큰 만료 (expiry_dt ≤ NOW()) | 400 | /auth/password/invalid?reason=EXPIRED 로 이동 |
| 토큰 사용완료 (token_sttus_code = 'USED') | 400 | /auth/password/invalid?reason=USED 로 이동 |
| 토큰 존재하지 않음 | 400 | /auth/password/invalid?reason=INVALID 로 이동 |

#### FID-00032 새 비밀번호 저장

#### 기능: [FID-00032] 새 비밀번호 저장

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | UPDATE |
| **API** | `POST /api/auth/password/reset` |
| **트리거** | [재설정 완료] 버튼 클릭 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| token | string | Y | URL query parameter에서 전달 |
| newPassword | string | Y | 새 비밀번호 |
| newPasswordConfirm | string | Y | 새 비밀번호 확인 |

**처리 로직**
```
1. 클라이언트 유효성 검증 통과 확인
2. API 호출 → 버튼 로딩 상태 전환
3. 서버 처리:
   a. tb_cm_password_reset_token에서 token_val 재검증 (PENDING + 미만료)
   b. tb_cm_member에서 해당 회원의 pswd_hash와 새 비밀번호 bcrypt 비교 → 동일하면 400 에러
   c. tb_cm_member UPDATE (pswd_hash = 새 비밀번호 bcrypt 해시, mdfcn_dt = NOW())
   d. tb_cm_password_reset_token UPDATE (token_sttus_code = 'USED', use_dt = NOW())
   e. tb_cm_refresh_token: 해당 회원의 모든 활성 RT UPDATE (revoke_dt = NOW())
   f. tb_cm_member_session: 해당 회원의 모든 활성 세션 UPDATE (invald_dt = NOW())
4. 완료 → 로그인 화면(PID-00006)으로 이동
```

**참조 테이블**
| 테이블 | 컬럼 | 용도 |
|:-------|:-----|:-----|
| tb_cm_password_reset_token | token_val | 토큰 재검증 |
| tb_cm_password_reset_token | token_sttus_code | 'PENDING' → 'USED' 상태 변경 |
| tb_cm_password_reset_token | use_dt | 사용 일시 기록 (NOW()) |
| tb_cm_password_reset_token | mber_id | 대상 회원 FK |
| tb_cm_member | pswd_hash | 기존 비밀번호 비교 + 새 bcrypt 해시 저장 |
| tb_cm_member | mdfcn_dt | 수정 일시 갱신 |
| tb_cm_refresh_token | revoke_dt | 해당 회원의 모든 활성 RT → NOW() (전 기기 로그아웃) |
| tb_cm_member_session | invald_dt | 해당 회원의 모든 활성 세션 → NOW() (전 세션 무효화) |

**에러 처리**
| 상황 | HTTP | 메시지 |
|:-----|:-----|:-------|

#### FID-00033 오류 유형별 안내 표시

#### 기능: [FID-00033] 오류 유형별 안내 표시

| 항목 | 내용 |
|:-----|:-----|
| **기능유형** | SELECT |
| **트리거** | 화면 진입 시 자동 실행 |

**Input**
| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|

**처리 로직**
```
1. URL에서 reason 파라미터 추출
2. reason별 메시지 및 버튼 상태 렌더링
   - EXPIRED: 링크가 만료되었습니다. 재발송을 요청해 주세요 + [재발송 요청] 활성

3. reason 파라미터 없거나 알 수 없는 값 → INVALID로 처리
```
