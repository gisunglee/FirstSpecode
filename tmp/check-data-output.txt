## 기능 개요
프로젝트의 생성, 수정, 삭제, 복사 기능을 제공한다. 프로젝트 생성자는 자동으로 OWNER 역할이 부여되며, 내가 속한 프로젝트만 목록에 조회된다. 삭제 및 복사 시 하위 데이터(멤버, 요구사항, 설계 등) 전체가 연쇄 처리된다.

## 메뉴 위치
- 사용자: GNB 좌측 프로젝트 드롭다운 > 프로젝트 목록
- 관리자: 프로젝트 목록 > 프로젝트 설정 (OWNER만 접근)

## 사용 대상 / 권한
| 구분 | 대상 | 접근 범위 |
|------|------|-----------|
| 프로젝트 목록 조회 | 모든 로그인 사용자 | 본인이 속한 프로젝트만 |
| 프로젝트 생성 | 모든 로그인 사용자 | 생성자는 자동 OWNER |
| 프로젝트 수정 | OWNER | 프로젝트명, 기간, 고객사 등 기본 정보 |
| 프로젝트 삭제 | OWNER | 하위 데이터 전체 삭제 (확인 필수) |
| 프로젝트 복사 | OWNER, ADMIN | 설정 및 하위 구조 복사 |

## 제공 화면 목록
| 화면명 | 설명 |
|--------|------|
| 프로젝트 목록 | 내가 속한 프로젝트 카드형 목록, 생성 버튼 |
| 프로젝트 생성 팝업 | 프로젝트명, 기간, 고객사 입력 후 생성 |
| 프로젝트 설정 | 기본 정보 수정, 삭제, 복사 기능 |
| 프로젝트 삭제 확인 팝업 | 삭제 전 경고 및 최종 확인 |

## 기능 상세
| 기능명 | 설명 | 비고 |
|--------|------|------|
| 프로젝트 목록 조회 | 본인이 멤버인 프로젝트만 조회, 카드형 UI | 프로젝트명, 고객사, 기간, 역할 표시 |
| 프로젝트 생성 | 프로젝트명(필수), 시작/종료일, 고객사 입력 | 생성 후 자동으로 OWNER 멤버십 부여 |
| 프로젝트 수정 | 기본 정보(프로젝트명, 기간, 고객사, 설명) 변경 | 변경 이력 자동 기록 |
| 프로젝트 삭제 | 하위 멤버, 요구사항, 설계 데이터 전체 삭제 | 삭제 확인 팝업 필수, 되돌릴 수 없음 |
| 프로젝트 복사 | 프로젝트 설정 + 하위 구조를 새 프로젝트로 복사 | "[복사] 원본명"으로 생성, 데이터는 미포함 |

## 업무 처리 순서
1. 사용자가 프로젝트 목록에서 "새 프로젝트" 클릭
2. 프로젝트명, 기간, 고객사 입력 후 생성
3. 생성 완료 시 프로젝트 목록으로 이동 (새 프로젝트 표시)
4. 프로젝트 설정에서 기본 정보 수정 가능
5. 필요 시 프로젝트 복사 또는 삭제 실행

## 제외 범위 / 제약 사항 / 협의 사항
- (제외) 프로젝트 아카이브(보관) 기능은 1차 범위에서 제외
- (제외) 프로젝트 간 데이터 이동/병합 기능은 제외
- (제약) 프로젝트 삭제는 OWNER만 가능하며, 하위 데이터 전체 삭제됨
- (제약) 프로젝트명은 중복 허용 (UUID로 구분)
- (협의) 프로젝트 복사 시 멤버 포함 여부는 추후 협의
