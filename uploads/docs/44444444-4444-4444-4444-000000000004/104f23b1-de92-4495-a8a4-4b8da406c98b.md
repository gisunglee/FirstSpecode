========================================
점검 기준 (6개 카테고리)
========================================

## 1. 영역-기능 항목 대조
- 영역 구성 항목에 있는 버튼·입력 필드에 대응하는 FID가 존재하는지
- FID가 있는데 영역 구성 항목에 해당 UI 요소가 없는지
- 예: 영역에 [삭제] 버튼 → 삭제 FID 없으면 🔴
- 예: FID에 "조회수 증가" 로직 → 영역에 조회수 항목 없으면 🟡

## 2. 기능(FID) 명세 완전성
- 기능유형이 적절한지 (SELECT/INSERT/UPDATE/DELETE/COMPOSITE)
- 서버 호출 기능에 API 경로가 있는지
- Input/Output이 처리 로직과 일치하는지
- 처리 로직의 단계가 논리적으로 완결되는지
- 에러 처리가 정의되어 있는지

## 3. 참조 테이블 정합성
- FID 처리 로직에서 사용하는 테이블/컬럼이 참조 테이블에 모두 나열되었는지
- 참조 테이블에 있는데 처리 로직에서 안 쓰는 테이블이 있는지

## 4. 권한 반영 여부
- 전체설계의 권한 정의에서 제한된 기능이 FID 에러 처리에 403으로 반영되었는지
- 영역 구성 항목에서 권한별 숨김/비활성화 처리가 명시되었는지

## 5. 화면 흐름 연결성
- 영역 내 이동 버튼(목록, 수정, 상세 등)이 전체설계의 화면 흐름과 일치하는지
- 전달 파라미터가 수신 화면에서 실제 필요한 값인지

## 6. API 경로 일관성
- 해당 영역 FID들의 API 경로가 RESTful 컨벤션을 따르는지
- 동일 리소스 CRUD 경로가 일관적인지
